# dosfm — база знаний проекта

Живой документ: решения, грабли, как запускать. Обновлять по ходу.

## Идея (одной строкой)
Браузерная DOS-консоль (js-dos) с DOS-файловым менеджером для управления файлами, которые персистентно
хранятся на бэкенде (FastAPI) и синхронизируются двусторонне.

## Архитектура
- **Бэкенд FastAPI** — рабочая папка `backend/workspace/`, API: `GET /api/tree`, `GET /api/download`, `POST /api/sync`.
  Все пути — строго внутри `workspace/` (защита от path traversal).
- **Фронт** — js-dos (DOSBox/WASM) + DOS-файловый менеджер. Мост: на старте заливаем файлы сервера в ФС
  эмулятора (`fsWriteFile`), по «Синхронизации» — читаем ФС (`fsTree`/`fsReadFile`) и шлём на `/api/sync`.

## Стек / окружение

- js-dos подключаем локальным vendor'ом (офлайн) — версия/ссылка появятся на M2.

## DOS-менеджер (лицензии)

  Volkov Commander — freeware, но не открытый. Финально выбрать на M2.

## Фичи (запрос пользователя, 2026-07-05)
- Переименование в **CMS-DOS** (title, шапка, ASCII-логотип в консоли через autoexec).
- **Аутентификация** (🔴 красная зона, ревью Дениса): `app/auth.py` — 1 пользователь, дефолт admin/admin,
  `must_change=True` (смена при первом входе), пароль pbkdf2+соль в `backend/data/auth.json` (chmod600, .gitignore),
  сессии в памяти, cookie `cmsdos_session` httponly. Эндпоинты `/api/{login,logout,me,passwd}`.
  Весь файловый API — под `require_auth`. Тесты `test_auth.py` (6). Итого бэк-тесты: **11**.
- **Кнопка «Полный экран»** — Fullscreen API на `#dos`.
- **DOS-команды через мост** (маркер-файлы + опрос `ci.fsTree`):
  - `uploads` (UPLOADS.BAT в PATH C:\CMSDOS) — из любой папки: пишет маркер `CMSUPREQ.TMP` с текущим каталогом →
    браузер открывает выбор файла → сохраняет на сервер (`POST /api/upload`) + пишет в ФС эмулятора (`fsWriteFile`).
  - `passwd` (PASSWD.BAT) — маркер `CMSPWREQ.TMP` → браузер открывает окно смены пароля (со старым паролем).
  Мост опрашивает ФС каждые 1.2с после события `ci-ready`.

## Грабли / решения
- Порты 8000/8080/3000 заняты чужими серверами в контейнере → dosfm на **5173**.
- Мост js-dos↔браузер: точный формат путей `fsTree`/`fsWriteFile` подтверждается только в браузере —
  возможна 1 правка после ручного теста (сервер-сохранение при этом надёжно, оно не зависит от ФС эмулятора).
