# CMS-DOS

Автор: **Семенцул Максим** (MaGz GoLd) · © 2026 · лицензия GPLv2

Браузерная **DOS-консоль (js-dos / DOSBox в WebAssembly)** для управления файлами.
Диск **C:** = серверная рабочая папка; изменения из DOS **автоматически** сохраняются на сервер.

## Возможности
- Вход по логину/паролю (по умолчанию **admin / admin**, смена обязательна при первом входе; далее — команда `passwd` в DOS).
- Диск C: — реальные файлы серверной папки (`backend/workspace`).
- **Автосохранение** изменений/создания/удаления на сервер (без кнопок, каждые ~2.5 с).
- DOS-команда **`uploads`** — загрузить файл с компьютера в текущую папку (из любого каталога).
- DOS-команда **`passwd`** — сменить пароль.
- Кнопка **«Полный экран»**.

## Требования
- Python 3.10+ (для FastAPI-бэкенда). Всё остальное (js-dos) уже внутри, интернет не нужен.

## Запуск
```bash
cd backend
python3 -m venv .venv
. .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 5173
```
Открой в браузере: **http://localhost:5173**  → войди `admin` / `admin` → задай новый пароль.

## Как это устроено
- `backend/` — FastAPI: аутентификация + API рабочей папки (`/api/tree`, `/api/download`, `/api/sync`, `/api/upload`).
  Рабочая папка — `backend/workspace` (или переменная `DOSFM_WORKSPACE`).
- `frontend/` — статика: `index.html`, `app.js`, локальный `vendor/` (js-dos, офлайн).
- Пароль хранится хешированным (pbkdf2+соль) в `backend/data/auth.json` (создаётся при первом запуске).

## Тесты (бэкенд)
```bash
cd backend && . .venv/bin/activate && python -m pytest -q
```

## Безопасность / прод
- Пароль: минимум **8 символов**; хеш pbkdf2-sha256 **600k** итераций + соль.
- Rate-limit на входе: 5 неудачных попыток за 5 мин на IP → 429.
- Сессия живёт 12 часов.
- **На HTTPS-проде** выставь `CMSDOS_COOKIE_SECURE=1` — cookie сессии не уйдёт по HTTP.
